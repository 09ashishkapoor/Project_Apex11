What MANTRINI offers

The app is built for focused practice, steady counting, and local record-keeping without requiring connectivity.

Guided counting modes

  • Manual counting for direct, tactile control.
  • Audio-guided counting for supported chanting flow.
  • Timed automatic chanting for uninterrupted practice sessions.

Four bundled deity paths

  • Current four bundled deity and mantra paths are available inside the app.
  • Practice flows are designed around devotional consistency rather than internet features.
  • Android remains the primary supported experience for this release channel.

Private local history

  • Session history is stored locally in Drift and SQLite on your own device.
  • Today totals, lifetime totals, malas of 108, recent sessions, and per-deity counts stay available offline.
  • Your practice records do not require a server account.

Backup and continuity

  • Export session backups as JSON text.
  • Import backups back into the app when needed.
  • Keep your own archive without handing data to a cloud service.

Trust and privacy by design

MANTRINI keeps the product boundary narrow. The absence of network-driven features is part of the design, not a missing upsell path.

No account layer

No backend, no login, no password reset flow, and no cloud profile to maintain.

No monetization traps

No payments, no subscriptions, and no ad surfaces interrupting practice.

No telemetry

No analytics, no social feed, and no hidden cloud sync transferring your practice data elsewhere.

Android install notes

  • On some phones, the first sideload install requires enabling installs from the browser or file manager that opened the APK.
  • For safety, use only APKs downloaded from this repository’s official GitHub release pages.
